# 0001. ブログ基盤に Astro を採用する

**決定**: Go 勉強ログのブログ基盤に **Astro（静的サイトジェネレータ）** を採用する。

## なぜ（コードを読んでも出てこない理由）

- デザイン自由度が高く、「技術ブログ＋遊び心＋おしゃれ」という見た目要件を素のCSSで作り込める。
- Markdown をそのまま記事化でき、Notion からの素材移行と相性がよい（運用は都度 Markdown を整形）。
- ローカルで即プレビューでき、編集→確認の反復が速い。
- 候補だった Hugo（Go製でテーマ題材と相性◎・高速）は、細かいデザイン表現の手間が Astro より大きいと判断。Next.js は動的要件が無く過剰。

## 却下した選択肢

- **Hugo**: 高速・Go製で題材と好相性だが、遊び心ある独自デザインの作り込みが重い。
- **Next.js**: 静的ブログには過剰（動的機能の予定なし）。
- **プレーンHTML/CSS**: 記事が増えると手作業が増える。
