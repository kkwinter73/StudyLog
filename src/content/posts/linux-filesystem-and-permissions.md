> 📚 シリーズ「Linux をちゃんと使う」(3 / 6)

「設定ファイルはどこ？ログはどこ？」を迷わないための **ディレクトリの使い分け**と、
ファイルへのアクセスを決める **パーミッション（rwx）** を押さえる。サーバー運用の基礎体力。

## 主要ディレクトリの使い分け

Linux はディレクトリの役割がだいたい決まっている（FHS という標準）。よく触る4つ。

| 場所 | 何を置く | 例 |
| --- | --- | --- |
| `/etc` | システム・アプリの**設定ファイル** | `/etc/nginx/nginx.conf` |
| `/var/log` | **ログ**（増えていくデータ） | `/var/log/syslog` |
| `/home` | 各ユーザーの**個人領域** | `/home/alice`（＝ `~`） |
| `/tmp` | 一時ファイル（**再起動で消えうる**） | ダウンロード途中の作業用 |

> 💡 「設定は `/etc`、ログは `/var/log`」をまず体に入れる。トラブル時に**最初に見る場所**がこの2つ。

## パーミッションの読み方（rwx）

`ls -l` の先頭の記号がパーミッション。**3文字×3組**で「誰が」「何を」できるかを表す。

```text
-rwxr-xr--   1 alice staff  ...  app
 │└┬┘└┬┘└┬┘
 │ │  │  └ その他(other)：r--  → 読みのみ
 │ │  └─── グループ(group)：r-x → 読み＋実行
 │ └────── 所有者(owner)： rwx → 読み書き実行
 └──────── 種別（- ファイル / d ディレクトリ）
```

| 記号 | 意味 | ファイル | ディレクトリ |
| --- | --- | --- | --- |
| `r` | 読み | 中身を読める | 一覧できる |
| `w` | 書き | 編集できる | 中に作成・削除できる |
| `x` | 実行 | 実行できる | **その中に入れる（cd）** |

## 数字表記と chmod

rwx は **r=4・w=2・x=1** の合計で数字にできる。3桁で owner/group/other を表す。

| 数字 | 意味 | よく使う対象 |
| --- | --- | --- |
| `755` | owner=rwx, group/other=r-x | 実行ファイル・ディレクトリ |
| `644` | owner=rw-, group/other=r-- | 普通のファイル |
| `600` | owner=rw-, 他=なし | 秘密鍵・機密設定 |

```bash
chmod 755 deploy.sh     # 数字で指定
chmod +x deploy.sh      # 実行権だけ足す（記号でも指定可）
```

## 所有者を変える chown

ファイルには **所有者(user)とグループ**がある。それを変えるのが `chown`。

```bash
chown alice file.txt          # 所有者を alice に
chown alice:devs file.txt     # 所有者 alice・グループ devs に
chown -R app:app /var/www/    # ディレクトリ以下まとめて（-R）
```

> ⚠️ 「Permission denied」の多くは**所有者かパーミッションのズレ**。`ls -l` で誰の何の権限かを
> まず確認するのが解決の近道。秘密鍵は `600` でないと SSH に拒否される（次回扱う）。

## まとめ

- 役割は決まっている: **設定=`/etc`、ログ=`/var/log`、個人領域=`/home`、一時=`/tmp`**
- パーミッションは **owner/group/other × rwx**。ディレクトリの `x` は「入れる」の意味
- 数字表記は **r=4 w=2 x=1** の合計。`755`(実行物)・`644`(普通)・`600`(秘密)が定番
- `chmod` で権限、`chown` で所有者を変える（再帰は `-R`）
- 「Permission denied」は `ls -l` で所有者・権限のズレを疑う
