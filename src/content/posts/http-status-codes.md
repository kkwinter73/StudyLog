> 📚 シリーズ「ネットワークをちゃんと使う」(5 / 5)

[接続トラブルの切り分け](/posts/connection-troubleshooting/)が「**繋がるか**」の話なら、こちらは
**繋がった後**の話。サーバーは応答ごとに **HTTPステータスコード**を返す。その読み方を押さえると、
API デバッグで「どっちのせいか」がすぐ分かる。

## まず5つのクラス

3桁の先頭の数字で大分類が決まる。**まず1桁目を見る**のがコツ。

| クラス | 意味 | ひとことで |
| --- | --- | --- |
| 1xx | 情報 | 処理継続中（普段あまり見ない） |
| 2xx | 成功 | うまくいった |
| 3xx | リダイレクト | 別の場所へ移動して、の指示 |
| 4xx | クライアントエラー | **こちらの要求**がおかしい |
| 5xx | サーバーエラー | **サーバー側**で問題が起きた |

> 💡 デバッグの肝は **4xx か 5xx か**。4xx は自分の送ったリクエストを、5xx はサーバー(やその先)を疑う。
> この一点だけで「どっちを調べるか」が決まる。

## 頻出コードの意味

実務でよく見るのはこのあたり。丸暗記より「クラスのどれか」と結びつけて覚える。

| コード | 意味 | よくある原因 |
| --- | --- | --- |
| 200 OK | 成功 | 正常 |
| 301 / 302 | 恒久 / 一時リダイレクト | http→https、URL変更 |
| 400 Bad Request | リクエストが不正 | JSON崩れ・必須パラメータ欠落 |
| 401 Unauthorized | 認証が必要/失敗 | トークン未添付・期限切れ |
| 403 Forbidden | 認証はOKだが権限なし | アクセス権限不足 |
| 404 Not Found | 対象が無い | URL/パス間違い |
| 500 Internal Server Error | サーバー内部エラー | アプリの例外・バグ |
| 502 / 503 | 上流不正 / 一時利用不可 | 背後のアプリ落ち・過負荷・再起動中 |

> ⚠️ **401 と 403 は別物**。401＝「あなたが誰か分からない(認証)」、403＝「誰かは分かるが許可がない(認可)」。
> 401ならトークン、403なら権限設定を見る。

## 4xx / 5xx で疑う場所

「繋がったのにエラー」の切り分けは、クラスで分岐させる。

```text
4xx → 自分のリクエストを見直す（URL・ヘッダ・ボディ・認証トークン）
5xx → サーバー側を見に行く（アプリのログ・例外、背後のサービス、負荷）
      502/503 は「LB は生きてるが背後のアプリが応答してない」典型
```

> 🧭 502/503 は[ロードバランサ](/posts/aws-networking-explained/)が前段にいる構成で頻出。
> LB 自体は応答するのでTCPは繋がるが、背後のアプリが落ちている/起動中だと 502/503 が返る。
> [グレースフルシャットダウン](/posts/deploying-go-apps/)が効くのはここ（切替時の 5xx を減らせる）。

## curl で実際に見る

ステータスコードは `curl` でそのまま確認できる。調査の基本動作。

```bash
curl -I https://example.com                       # ヘッダだけ取得（先頭にコード）
curl -s -o /dev/null -w "%{http_code}\n" URL       # コードだけ表示
curl -v https://example.com                        # 詳細（リダイレクトの連鎖も見える）
```

> 💡 このブログの公開確認で使っている `-w "%{http_code}"` がまさにこれ。200 なら成功、
> 3xx ならリダイレクト先を、4xx/5xx なら上の表で原因の当たりを付ける。

## まとめ

- ステータスコードは**繋がった後**のサーバーの返事。まず**1桁目(クラス)**を見る
- **2xx成功 / 3xxリダイレクト / 4xxクライアント / 5xxサーバー**
- デバッグの肝は **4xx(自分のリクエスト) か 5xx(サーバー側) か**
- 401(認証)≠403(認可)、404はパス間違い、502/503は背後のアプリが応答してないサイン
- `curl -I` や `-w "%{http_code}"` でコードを確認し、クラスで調べる場所を決める
