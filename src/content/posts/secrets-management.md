> 📚 シリーズ「CI/CD と IaC」(3 / 3)

シリーズ最後は、[CI/CD](/posts/cicd-github-actions/) でも [IaC](/posts/iac-terraform-basics/) でも必ず関わる横断テーマ。
APIキーやDBパスワードなどの**シークレットを、コードに直書きしてはいけない理由**と、ではどこに置くかを整理する。

## なぜハードコードがダメか

「とりあえず動くから」とコードに直書きすると、後で確実に困る。

- **Git履歴に永久に残る**: 後で消しても、過去のコミットを遡れば見える。一度入れたら漏れたも同然
- **見える人全員に漏れる**: リポジトリにアクセスできる人・CIログ・公開時、すべてに晒される
- **ローテーションが地獄**: キーを変えるたびにコードを直して再デプロイが要る
- **環境を分けられない**: 開発と本番で別のキーを使い分けられない

> ⚠️ 「private リポジトリだから大丈夫」も危険。公開設定の事故、退職者のアクセス、外部連携で漏れる。
> **秘密はそもそもコードに置かない**のが原則。

## どこに置くか

コードからは「外から渡される」前提にし、実体は専用の場所に置く。

| 置き場所 | 用途 |
| --- | --- |
| 環境変数 | 実行時にアプリへ注入（[12-factor](/posts/deploying-go-apps/)の基本） |
| GitHub Actions Secrets | [CI/CD](/posts/cicd-github-actions/)パイプライン内で使う秘密 |
| AWS Secrets Manager / SSM | 本番アプリが参照する秘密（[AWS⑤](/posts/aws-security-config-explained/)） |

```yaml
# GitHub Actions：secrets はログにも出ない形で渡る
steps:
  - run: ./deploy.sh
    env:
      API_KEY: ${{ secrets.API_KEY }}   # リポジトリ設定で登録した値
```

```go
// アプリ側は環境変数から読むだけ。値はコードに無い
apiKey := os.Getenv("API_KEY")
```

> 🧭 .NET の `appsettings.json` に秘密を書かず、User Secrets や環境変数・Key Vault を使うのと同じ。
> 「設定ファイルやコードに秘密を焼かない」は言語・基盤を問わない鉄則。

## うっかり混入を防ぐ

- **`.gitignore`** に `.env` など秘密を含むファイルを必ず入れる（コミット対象から外す）
- [Terraform の state](/posts/iac-terraform-basics/) には秘密が平文で入りうる。state はGitに入れず、S3等の安全な場所へ
- **漏れたら即無効化**。消すより先に、そのキー自体をローテーション（再発行）するのが最優先

> ⚠️ 「コミットしてから消す」では遅い。Git履歴に残るので、混入に気づいたら**そのキーを無効化**してから対処する。

## まとめ

- ハードコードは **Git履歴に永久に残る・全員に漏れる・ローテーション困難**。原則やらない
- 秘密はコードから**外（環境変数など）に出し**、アプリは環境変数で読むだけにする
- 置き場所: 実行時は**環境変数**、CIは**GitHub Secrets**、本番は**Secrets Manager/SSM**
- `.gitignore` で `.env` を除外。**Terraform state** も秘密を含むのでGitに入れない
- 漏れたら消すより先に**キーを無効化（ローテーション）**する
