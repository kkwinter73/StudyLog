攻撃者が本番の攻撃に入る前に必ずやる「偵察」の代表例、ポートスキャンを整理する。
結論を先に言うと、**開いているポート＝攻撃面**なので、守る側の仕事は「不要なポートを閉じて、
自分でスキャンして確認する」——これに尽きる。

## 攻撃の仕組み — スキャンは殴る前の下見

サーバは[ポート番号ごとにサービスを待ち受ける](/posts/ports-and-tcp/)。攻撃者はまず、
標的のどのポートが開いているかを機械的に総当たりして、そこから**攻め口**を探す。これが偵察。

- **開いているポート**が分かれば、そこで何のサービスが動いているか（SSH・HTTP・DB など）が推測できる
- サービスが分かれば**バージョン**を特定でき、そのバージョンに既知の脆弱性がないか調べられる
- つまりスキャンの成果物は「どこを、何で攻めればよいかの地図」。攻撃はその後に来る

スキャン方式にはいくつかあり、代表的なのは次の2つ。

| 方式 | やること | 特徴 |
| --- | --- | --- |
| **TCP connect** | 普通に3ウェイハンドシェイクを最後まで完了させる | 確実・権限不要だが、接続がログに残りやすい |
| **SYN スキャン** | SYN だけ送り、応答（SYN-ACK / RST）で開閉を判定して RST で中断 | 接続を完了しないので「ステルス」。管理者権限が要る |

> 💡 SYN-ACK が返れば「開」、RST が返れば「閉」、無反応なら「フィルタされている（FW あり）」と推測される。
> 応答の有無そのものが情報になる。

代表的なツールが `nmap`。数百のポートを短時間で調べ、サービスやバージョンまで当ててくる。

## 対策の原則 — 攻撃面を減らす

守りの基本は「攻撃者に見せる面を最小化する」こと。開いていないポートは攻撃できない。

- **使っていないポートを閉じる**: 動かしていないサービスは止める。開いている＝攻撃対象、と考える
- **デフォルト拒否**: ファイアウォールや[セキュリティグループ](/posts/aws-security-groups/)は
  「原則すべて拒否、必要なものだけ許可」で組む。許可リスト方式にする
- **内部向けサービスを外に晒さない**: DB・管理画面・監視用ポートは、公開せず社内網や VPN 経由に限定する
- **動かすサービスは patch を当て続ける**: 開けざるを得ないポート（80/443 など）は、
  そこで動くソフトを最新に保つ。偵察で「古いバージョン」が見えた瞬間に狙われる

> ⚠️ 一番危ないのは「気づかず開いていたポート」。誰も使っていない管理用ポートや、
> 検証で開けて閉じ忘れたポートが、攻撃者にとって最良の入口になる。

> 🧭 開発現場の感覚で言えば「使わない機能フラグや公開エンドポイントは消す」のと同じ。
> 露出を足すのは簡単だが、消し忘れが事故を生む。攻撃面も同じ発想で削る。

## 実務での関わり方 — 同じツールで自分を点検する

偵察ツールは攻撃者だけのものではない。**守る側が自分のホストにスキャンをかけて、
「外から何が見えているか」を確認する**のが実務での使い方（自分の資産に対してのみ実施する）。

まず、そのホストで実際に何が待ち受けているかは `ss` で確認できる。

```bash
# LISTEN 状態の TCP ポートとプロセスを一覧（-t TCP / -l LISTEN / -n 数値 / -p プロセス）
ss -tlnp
# 旧環境なら netstat でも同じことができる
netstat -tlnp
```

次に、外から見た姿を `nmap` で自分のホストに対して確認する。

```bash
# 自分のホストの開放ポートとサービス/バージョンを調べる（-sV でバージョン検出）
nmap -sV 203.0.113.10
```

- ここで**意図せず開いているポート**が見えたら、それが減らすべき攻撃面
- クラウドなら、結果を[セキュリティグループ](/posts/aws-security-groups/)のルールと突き合わせる。
  「SG では閉じているつもりなのに nmap から見える」なら設定ミス、逆もまた点検対象
- 大量のスキャンが外から来ていること自体が兆候になる。極端なものは
  [DoS/DDoS](/posts/dos-ddos/) の前触れのこともあるので、アクセスログの監視とセットで見る

> ⭐ 攻撃者と守る側は同じ地図を見ている。違うのは、守る側は**先に見て塞げる**こと。
> 定期的に自分をスキャンして、開いている面が意図どおりかを確かめるのを習慣にする。

## まとめ

- ポートスキャンは攻撃前の**偵察**。開いているポートとサービス・バージョンから攻め口を探る行為
- 方式は **TCP connect（確実・ログに残る）** と **SYN スキャン（ステルス・要権限）** が代表
- 守りの原則は**攻撃面を減らす**こと: 不要ポートを閉じ、FW/SG はデフォルト拒否、内部サービスを晒さず、patch を当て続ける
- 守る側は**同じ nmap で自分をスキャン**し、`ss`/`netstat` の LISTEN と突き合わせて意図しない露出を潰す
- クラウドでは結果を**セキュリティグループのルール**と照合するのが実務の定番
